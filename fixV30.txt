## Bug 1：我的最愛加入食物不會進入今日紀錄

**問題位置：** `src/js/dashboard-modals.js` - `addFoodFromDatabase()` 函數（第 116 行）

**根本原因：**
當從「我的最愛」呼叫 `addFoodFromDatabase(id, name, calories, protein, carbs, fat, isFavorite=true)` 時，
API 呼叫的 body 只傳了 `{ barcodeId: id, isFavorite }`，
沒有傳 `name, calories, protein, carbs, fat`。

後端 `POST /api/food/add-from-database`（`src/routes/food.js` 第 337 行）的邏輯是：
1. 先判斷有沒有 `name && calories !== undefined` → 直接用傳入資料
2. 再判斷 `barcodeId && isFavorite` → 去 FavoritesDB 查

當 sort 為 `most_used` 時，回傳的資料 `isStats: true`，
此時呼叫的是 `addFoodFromStats(calories, protein, carbs, fat, name, imagePath)` — 這條路徑是正確的。

但當 sort 為 `recent` 或 `name` 時，呼叫的是 `addFoodFromDatabase(food.id, ..., true)`，
這時後端去 FavoritesDB 查 `barcodeId`，但 `food.id` 是 `favorites` 表的 id，應該是正確的。

**請確認並修復：**
在 `addFoodFromDatabase` 函數中，當 `isFavorite=true` 時，
同時把 `name, calories, protein, carbs, fat` 也傳入 body，
讓後端優先走第一條邏輯（直接用傳入資料），避免依賴 barcodeId 查找失敗：

```js
body: JSON.stringify({ barcodeId: id, isFavorite, name, calories, protein, carbs, fat })
```

另外，`addFoodFromDatabase` 呼叫成功後要呼叫 `window.loadFoodLog()` 刷新今日紀錄，
請確認這行有被執行（目前有，但需確認是否有 early return 跳過）。

## Bug 2：歷史統計數據不見了

**問題位置：** `src/services/database.js` - `DailyProgressDB.upsert()` 第 818 行

**根本原因：**
`upsert` 的 SQL ON CONFLICT 更新區塊缺少 `goal_calories` 的更新：

```sql
ON CONFLICT(user_id, date) DO UPDATE SET
  total_calories = excluded.total_calories,
  total_protein = excluded.total_protein,
  total_carbs = excluded.total_carbs,
  total_fat = excluded.total_fat,
  updated_at = CURRENT_TIMESTAMP
  -- ❌ 缺少 goal_calories = excluded.goal_calories
```

但更嚴重的問題是：歷史統計顯示「尚無歷史資料」，
原因是 `DailyProgressDB.findByUserId` 只查 `daily_progress` 表，
而 `daily_progress` 只有在新增或刪除食物時才寫入，
如果使用者沒有觸發 upsert（例如重啟後沒有飲食記錄），資料就不存在。

**請執行以下修復：**

1. 修復 `DailyProgressDB.upsert` 加入 `goal_calories = excluded.goal_calories`

2. 在 `GET /api/progress/history`（`src/routes/progress.js` 第 129 行）中，
   改為同時查詢 `food_logs` 表做聚合，不只依賴 `daily_progress`：
   若 `daily_progress` 沒有某天的資料，就從 `food_logs` 即時計算補上，
   確保有食物記錄的日期都能顯示在歷史統計中。

   修改 `DailyProgressDB.findByUserId` 或在 progress route 中加入 fallback 查詢：
```sql
   SELECT 
     date,
     SUM(calories) as total_calories,
     SUM(protein) as total_protein,
     SUM(carbs) as total_carbs,
     SUM(fat) as total_fat
   FROM food_logs
   WHERE user_id = ? AND date BETWEEN ? AND ?
   GROUP BY date
   ORDER BY date DESC
```
